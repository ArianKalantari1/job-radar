# job-radar — Copilot Agent Prompt Package
## How to use this folder

Each file is one task for GitHub Copilot Agent mode.
**Feed them one at a time, in order.** Do not feed multiple prompts at once.

After each prompt:
1. Run the **Validation** steps listed at the bottom of the prompt
2. If it passes → move to the next prompt
3. If it fails → paste the error back to Claude and ask for a fix prompt

## Things Copilot cannot do (you handle these)

1. **Slack app creation** — go to https://api.slack.com/apps, create the app, get the Bot Token. Instructions in `prompt_07_ngrok_cron.md`.
2. **ngrok / Cloudflare Tunnel** — install and start the tunnel. Instructions in `prompt_07_ngrok_cron.md`.
3. **Fill in `.env`** — copy `.env.example` to `.env`, add your Anthropic API key, Slack token, channel ID.
4. **Upload your resume PDF** — Phase 3 (not in this package yet).
5. **Write `prompts/ari_rules.md`** — your cover letter structural rules. Phase 4 (not in this package yet).

## After all 8 prompts pass validation

Your system does:
- Scrapes 5 role families (DA, AE, DS, PM-AI, PM) twice daily
- Scores every job against all 5 profiles
- Sends role-grouped Slack digest with swipe buttons
- Swipes update job status in real time
- Kanban board with drag-drop, role filters, dismiss with reason

## Warning: what to protect

These functions in `scraper.py` are stable and should NOT be modified by Copilot:
- `init_db()` — only add migrations, never restructure
- `upsert_jobs()` — fingerprint dedup logic is battle-tested
- `fetch_description()` — selectors took effort to get right
- `enrich_descriptions()` — leave alone

If Copilot rewrites any of these unprompted, **undo and try a more constrained prompt**.
